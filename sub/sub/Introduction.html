Introduction

An Introduction to Syllable

The basic vision of Syllable is simple: A reliable and easy-to-use desktop operating system for home and small office users.

First off, here's what Syllable is not :

  1. Syllable is not competing with Windows or Linux. Syllable's better than a mere copycat; Syllable forges its own path.
  2. Syllable is not a platform for outrageous, experimental operating system ideas. You should be comfortable in Syllable, not bewildered by complex new paradigms.
  3. Syllable is not exclusively for the experts. It's meant for regular folks like you and me, while reserving plenty of power behind the scenes for those with the knowledge.
  4. Syllable is not bloated. It can boot in under 20 seconds on even old hardware.

So, Syllable is intended to be a cool replacement for Windows or Linux on your desktop.

Cool Features

What does Syllable give you that can't be found elsewhere?

First off, operating systems like Windows and MacOS have lots of legacy code to ensure that old software works properly. Syllable, on the other hand, isn't weighted down by the code needed to support decades' worth of old software. This helps make Syllable responsive and lightweight.

Secondly, Syllable uses a cool file system called AFS, which can store lots of common data (metadata) about files and lets you make searches on them like a database.

Thirdly, Syllable is POSIX-compliant. This means that Syllable can run a lot of tried-and-true Unix software, without needing tons of code to support it.

Forthly, Syllable boots very quickly even on old computers and has low RAM requirements.

For developers, Syllable provides a clean, completely object-oriented programming API that's a dream to program in.

Syllable is currently being developed, but is relatively stable and mature. A number of useful Syllable applications are already available, and Syllable supports a wide range of common hardware device, including video, network, and sound cards from manufacturers such as Intel, AMD, 3Com, nVidia, 3DFX, Matrox, and Creative. If you would like to see a more comprehensive list of supported hardware, see here: http://www.syllable-hardware.info/

Syllable History

The Syllable project dates back to July 2002, but the story begins well before that.

The history of Syllable really begins with an operating system named BeOS, which was developed throughout the 1990's by a small company named Be made up primarily of former Apple employees who were dissatisfied with the direction of Apple. They developed BeOS as a modern, powerful speed demon of an operating system that was particularly suited for heavy multimedia applications (you should have seen a BeOS machine running thirty simultaneous videos, smooth as butter).

Though Be was eventually bought out by Palm and BeOS was liquidated, it inspired a young programmer named Kurt Skauen in his efforts to create a hobby operating system. In the late 90's and early 00's, he created AtheOS, borrowing heavily from BeOS' file system and programming API.

After a few years, Kurt abandoned AtheOS to pursue other dreams. In July 2002, a team of devoted AtheOS enthusiasts forked AtheOS and made Syllable.

Since then, the Syllable development team has been working to polish Kurt's work and turn it into a general-purpose operating system that anyone can use.

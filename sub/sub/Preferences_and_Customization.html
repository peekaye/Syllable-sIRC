Preferences and Customization

Appearance Preferences

This preferences window lets you change the decor of Syllable to suit your own tastes. There are seven decors available:

  • Beish
  • Drow
  • Mulvane
  • Photon
  • Red
  • Win98
  • amiga

To change the decor, click the Decor drop-down, select a Decor, and click the Apply button.

You can also change the Themes within each Decor. Each Theme specifies a number of colors for each element of the Syllable user interface. The following elements are available:

  • Application – The background color for each application window
  • Highlight
  • Shadow
  • Active Window
  • Window Border
  • Menu Text
  • Menu Highlight Text
  • Menu Background
  • Menu Highlight
  • Scrollbar Background
  • Scrollbar Knob
  • Listview Tab
  • Listview Tab Text

To change the theme, select an element from the drop-down list, move the Red, Green, or Blue sliders or change the text fields, and click the Save Scheme button.

To delete a theme, select it in the Theme drop-down list and click the Delete Theme button.

Font Preferences

The Font Preferences window lets you choose the fonts used throughout Syllable. Syllable uses fonts in five different ways:

  • Normal – The default font
  • Bold – A bold version of the normal font
  • Fixed – Used when displaying code or other text that requires a fixed-width font
  • Window – Used for window titles
  • Tool Window

To change a font, click the appropriate drop-down menu, then click the Apply button. To change a font's size, type the new font size in the font size field next to the appropriate font, then click the Apply button.

By default, Syllable anti-aliases text, which smooths out the jagged edges of text. Anti-aliasing looks odd on some monitors.

To change Syllable's anti-aliasing policy, click the Anti-Aliasing checkbox underneath the Tool Window field and click the Apply button.

Keyboard Preferences

The keyboard preferences window lets you change the keyboard layout and certain keyboard options.

The Initial Delay is the amount of time that Syllable will wait after a key is first pressed to begin repeating the key. If this is set to a long period of time, Syllable will wait a long time before repeating the key.

The Repeat Delay is the amount of time that Syllable will wait to repeat a key once the key has begun repeating.

The following keyboard layouts are currently suppled with Syllable:

  • American
  • Belarusian
  • Belgian
  • British
  • Danish
  • Dvorak
  • Faeroese
  • French
  • German
  • Hungarian
  • Italian
  • Norwegian
  • Polish
  • Russian
  • Spanish
  • Swedish
  • Turkish_F
  • Turkish_Q
  • Ukrainian

To change the keyboard layout, select a new layout from the Keyboard Layout list, then click the Apply button.

To test your new keyboard preferences, click the Apply button and type in the Test Area field. Some characters won't display in the Test Area field owing to whichever font it is using.

Media Preferences

The Media Preferences window lets you change your preferences for a wide variety of media that is available in Syllable.

First, select the media that you want to affect in the list on the left-hand side of the window.

If the media has volume controls, you'll see two sliders for each control, as well as a Lock checkbox. If the Lock checkbox is checked, each slider will be set to whatever value the other slider is set to. If the Lock checkbox is unchecked, you can set each slider to different values.

If a media is set as a default output (for audio or video), a green checkbox will be displayed next to that media in the list.

To set a media as the default output for audio or video, select it in the list, then click the Default video output or Default audio output checkboxes.

The Video Overlay Output option won't work with some video cards, so select Video Bitmap Output if this is the case (Video Overlay Output will perform better, so use it when possible). Use Direct OSS if the Media Server option is is unreliable for you.

Clicking the Controls... button should display a window of general media controls.

Screen Preferences

The Screen Preferences window lets you control the resolution, colour depth, and refresh rate of this workspace or any workspace.

(Syllable maintains twelve workspaces, which you can think of as different work areas. If you launch a window in one workspace, you can switch to another workspace and it will be clear of the window in the first workspace. Multiple workspaces let you organize windows into different areas. To switch to workspace 2, press Alt F2; to switch to workspace 3, press Alt F3, and so forth.)

NOTE: You must be logged in as root to change screen preferences.

The Resolution controls how many pixels Syllable will display on the screen. A higher resolution will fit more on the screen, but everything on the screen will be smaller. Also, some older video cards and monitors do not support higher resolutions. You can choose from the following resolutions: 640x480, 800x600, 1024x768, 1152x864, or 1280x1024.

The Colour Depth controls the number of colors Syllable will use for each pixel. A higher colour depth will give you richer colors, but might slow down your computer somewhat. You can choose from the following colour depths: 15 Bit, 16 Bit, or 32 Bit. 16 Bit is generally the best option.

The Refresh Rate controls how many times per second Syllable refreshes the display. A higher refresh rate is usually easier on the eyes, but some older video cards and monitors do not support higher refresh rates. A high refresh rate could permanently damage your video card! You can choose from the following refresh rates: 60 Hz, 75 Hz, or 100 Hz. Leave it on 60Hz if you've got an LCD.

You can use the Apply to drop-down menu to select whether to apply your changes to This workspace or All workspaces.

When trying a setting for the first time, go to a new workspace (press ALT F12 for example) and use the This workspace option. This way, if you choose a setting that corrupts the screen or locks the computer up, you can recover the system from it. Be careful of using 32 Bit colour depth with the Matrox driver at this time, as it may cause lock ups.

To make changes, change any of the fields, then click the Apply button. To reset all fields to what they were when you first opened the Screen Preferences windnow, click the Undo button. To reset the fields to their default values of resolution 640x480, colour depth 15 Bit, and refresh rate 60 Hz, click the Default button.

Changing the Background

If you want to use your own image as your desktop background, copy the image to the Pictures folder in the Documents folder in your Home folder. Double-click the Preferences icon on the desktop, then double-click the Desktop icon in the Preferences window. Select the image you want to use as your background, and click the Apply button. The image you selected appears as the background image.

Images can be in JPEG, GIF, or PNG format.

Changing a Programs Icon

Changing a Program's Icon

To change a program's icon, you'll need to use the command-line tool "rescopy".

A program can have up to two PNG icons associated with it, one 24x24 pixels in size, and the other 48x48 pixels. These icons are stored as resources within the program's executable file.

To change a program's icon, you first need to know if there are icons already associated with it. Type the following in a terminal window: 

rescopy -l <application>

A list of all resources associated with the file is displayed. If there are no entries for "icon24x24.png" or "icon48x48.png" in the listing, the program has no icons associated with it, so you'll need to add your icons.

Adding an icon

To add an icon to a program, name the icon "icon24x24.png" if it's a 24x24 pixel icon, or "icon48x48.png" if it's a 48x48 pixel icon. To add a 24x24 pixel icon, enter the following in the terminal: 

rescopy -a <application> icon24x24.png

For a 48x48 pixel icon, enter the following: 

rescopy -a <application> icon48x48.png

You're done!

Replacing an icon

If the program already has associated "icon24x24.png" or "icon48x48.png" resourcess, you'll need to replace them by typing the following in a terminal window (assuming a 24x24 pixel icon): 

rescopy -r <application> icon24x24.png icon24x24.png

This instructs rescopy to replace the "icon24x24.png" resource with the "icon24x24.png" file you created. Replace "24x24" with "48x48" in the above example to replace the 48x48 pixel icon.

That's it!

More Information

Syllable uses the Bluecurve icons for many of its icons. You can download the Bluecurve icons for free; anybody can use them in any application.

For more information on rescopy, type the following in a terminal window: 

rescopy --help

NOTE: If rescopy quits with the error 'sh: objectcopy not found', install the Developer's Delight package

Fixing screen resolution

Problem: In the settings dialog of the Screen I set the resolution to 1280x1024, and now there appears a dialog on my monitor that says the resolution is out of limit and I can do nothing after booting Syllable.

Solution 1:

Press Alt F12. If the screen still doesn't display, go to Solution 2. If it does, open up the screen preferences, click Default, and change Apply To to All workspaces and click Apply.

Solution 2:

You'll need your Syllable installation CD to hand.

Boot from the CD and quit the installation script. This will leave you with a shell running in a single ATerm.

Mount your existing Syllable partition. I'll assume that your Syllable installation exists on /dev/disk/ata/hda/0, but you'll need to change this to suit your system. To mount your Syllable partition, type:

mkdir /syllable
mount /dev/disk/ata/hda/0 /syllable

You now need to edit a single configuration file to restore your display. The appserver configuration file can be found at:

/syllable/atheos/sys/config/appserver

Open it in AEdit and you'll see that the first line of the config file is something like:

Desktop 00 = 1280 1024 80.00 50.00 70.00 80.00 60.00 RGB16 ""

Change that line to match the others, e.g.:

Desktop 00 = 640 480 80.00 50.00 70.00 80.00 60.00 RGB16 ""

Save the file. Now reboot (remove the CD) and Syllable will reboot back into a 640x480 display, and you can try changing your display settings again.

Note
Each line in the file consists of the following fields: Desktop number, X, Y, horizontal position, vertical position, horizontal size, vertical size, refresh rate, colorspace, and backdrop.

Turning off Numlock

Question: How can numlock be turned on at boot time?

At the moment, the only way to do it is to edit the keymap you are using. To do that, follow these instructions:

I'll assume you're using the American keymap. Copy it someplace you can work on it, e.g.:

cp /system/keymaps/American ~/

Now run keymapconv:

cd ~
keymapconv American American.txt

Open American.txt in AEdit & change the line "IntialKeyCaps=" to "InitialKeyCaps=NumLock" Save the changes. Re-run keymapconv:

rm American
keymapconv American.txt American

Put the keymap back in /system/keymaps/

mv American /system/keymaps/

On your next reboot, the numlock won't be switched off by default.

Creating a Decorator

To create a decorator:

  1. Get the appserver source
  2. Dive into system/sys/appserver/appserver/drivers/decorators
  3. Copy one of the dirs
  4. Hack away! :-)

It's far easier to make a base a new decorator on existing code than to write from scratch. It's recommended that you try any decorator other than the Beish decorator, which is messy in places.

The Win98 and Amiga decorators are fairly easy to play around with. All of the decorators are built with vanilla drawing functions in C++; there are no pixmaps involved.

Essentially, you have a function which looks out for collisions (ie mouse over corners), a function which return button presses, various functions to draw the buttons, and a layout function which orders it all. There's more to it than that – but they're easy to get started with.
